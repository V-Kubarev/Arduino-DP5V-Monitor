## Основная цель

Превратить советский армейский дозиметр ДП-5В в современный цифровой дозиметр-радиометр с точными измерительными возможностями, статистическим анализом и интуитивным LCD-интерфейсом — сохраняя при этом образовательную ценность и экспериментальную гибкость.

---

## Философия проекта

### 1. **Образование прежде всего**
Проект ставит понимание выше "черного ящика". Каждый алгоритм документирован, объяснен и сделан прозрачным. Пользователи должны понимать *почему* что-то работает, а не только *что* оно работает.

### 2. **Научная точность**
Несмотря на самодельный характер, измерения должны быть статистически обоснованными:
- Корректная компенсация мертвого времени
- Статистика Пуассона для оценки погрешности
- Калибровка по эталонным приборам
- Прозрачные ограничения и допущения

### 3. **Практическая полезность**
- Одна кнопка для простоты управления
- Четкий, читаемый LCD-дисплей
- Звуковая/световая индикация импульсов
- Два режима отображения (текущий + накопленный)
- Автоматическая адаптация к уровню радиации

### 4. **Открытая разработка**
- Полная доступность исходного кода
- Детальная документация
- Модульная архитектура для экспериментов
- Контроль версий и журнал изменений

---

## Ключевые алгоритмы

| Алгоритм | Назначение | Ключевое новшество |
|-----------|---------|----------------|
| Аппаратные прерывания | Подсчет импульсов | Ни одного пропущенного импульса, даже при высоких скоростях |
| Программное мертвое время | Фильтрация сигнала | Исключение ложных срабатываний от "звона" импульсов |
| Динамическое скользящее окно | Усреднение CPS | Адаптация размера окна (5с/30с) к уровню радиации |
| Компенсация мертвого времени | Физическая коррекция | Математическая компенсация времени восстановления трубки |
| Статистика Пуассона | Оценка погрешности | Научно обоснованная оценка неопределенности |
| Интегрирование дозы | Накопление дозы | Непрерывный подсчет суммарной полученной дозы |

## Технический стек

- **Микроконтроллер:** ATmega328P (Arduino Uno/Nano)
- **Язык программирования:** C++ (Arduino-совместимый)
- **Дисплей:** LCD 16×2 с I2C
- **Ввод:** Одна кнопка с детекцией короткого/длинного нажатия
- **Вывод:** Пьезоизлучатель, светодиод, LCD
- **Источник сигнала:** Аудиовыход ДП-5В (с согласующей схемой)

## Принципы разработки

1. **KISS (Keep It Simple, Stupid)** — Избегайте ненужной сложности
2. **Документируй в процессе** — У каждой функции есть назначение, каждое назначение объяснено
3. **Тестируй на реальном железе** — Теория должна работать на реальных экземплярах ДП-5В
4. **Уважай аппаратуру** — Правильное согласование, никакого повреждения винтажного оборудования
5. **Научная строгость** — Измерения должны быть обоснованными

## Критерии успеха

1. **Точность:** ±20% от эталонного прибора после калибровки
2. **Стабильность:** Плавные показания на стабильном фоне
3. **Отзывчивость:** <10 секунд на обнаружение значительного повышения
4. **Удобство:** Интуитивно понятно для новичка
5. **Документация:** Полное описание всех алгоритмов
6. **Надежность:** Возможность круглосуточной непрерывной работы
